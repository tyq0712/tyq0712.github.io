基础内容#

实在太基础了拎出来单丢一节了

单调栈, 单调队列#

单次操作复杂度 $\mathcal O(n)$, 均摊 $\mathcal O(1)$

多用于某些问题的优化, 这类问题多类似贪心, 可以排除一些无用解.

Hash#

多用于 $\mathcal O(1)$ 计算某些难以 $\mathcal O(1)$ 计算是否相等的数据.

字符串的最强算法 : SAM $\times$ Hash $\sqrt{}\sqrt{}\sqrt{}$

字符串的 Hash 可以简单的差分.

同样可以支持对树做 $Hash$.

并查集#

维护集合的常用数据结构, 无法维护时考虑 LCT.

船新科技: 支持删除#

具体方法: 删除的时候新建一个空节点替代将被删除的节点即可. (惰性删除)

路径压缩的适用范围#

路径信息不重要的题目, 或者在压缩时可以快速并且不破坏的维护路径上的信息.

启发式合并与按秩合并#

启发式合并: 考察即将合并的两个集合的大小, 永远将小的集合合并到大的上 (如果能在 $\mathcal O(|\min\{ |S_1|, |S_2| \}|)$ 内完成合并, 即可实现 $O(n\log n)$ 时间复杂度的合并.)

按秩合并: 仅在并查集上能完成 $O(\log n)$ 时间复杂度. 按照深度, 将深度小的合并到深度大的上, 多用于路径压缩无法使用的场景. (例如可持久化并查集)

堆#

支持插入删除的维护最大值. 某些支持一些额外操作.

左偏树 (可并堆)#

定义某个节点的 $dist$ 是其到最近叶子的距离, 左偏树在满足堆性质的基础上满足自己的 $dist$ 为右儿子的 $dist$ 值增加 $1$(即左儿子的 $dist$ 不大于右儿子的 $dist$). 左偏树的左链的长度是 $\mathcal o(\log n)$, $\mathcal O(n)$ 的.

考虑解决两个左偏树的合并:

考虑使用此操作支持其他操作:

int newHeap(int val) : 直接支持即可, $\mathcal O(1)$

void insert(int x, int val) : merge(x, newHeap(val)); 即可, $\mathcal O(\log n)$

void del(int x) : merge(tr[x][0], tr[x][1]) 后从下向上考虑, 每当 $dist$ 性质被破坏时, 暴力维护即可, $\mathcal O(\log n)$.

对堆中的所有对象全体操作类: 凡是不改变相对大小(加减乘正数), 或者做 $\mathcal O(k)$ 次操作只改变 $\mathcal O(C)$ 次, 并且这种改变与当前堆中的数无关 (例如乘负数, 乘 $i$ (复数按照模长比较).).

「SCOI2011」棘手的操作#

题面#

有 $N$ 个节点, 标号从 $1$ 到 $N$, 这 $N$ 个节点一开始相互不连通. 第 $i$ 个节点的初始权值为$a_i$, 接下来有如下一些操作:

  • U x y 加一条边, 连接第 $x$ 个节点和第 $y$ 个节点.
  • A1 x v 将第 $x$ 个节点的权值增加 $v$.
  • A2 x v 将第 $x$ 个节点所在的连通块的所有节点的权值都增加 $v$.
  • A3 v 将所有节点的权值都增加 $v$
  • F1 x 输出第 $x$ 个节点当前的权值。
  • F2 x 输出第 $x$ 个节点所在的连通块中, 权值最大的节点的权值.
  • F3 输出所有节点中, 权值最大的节点的权值.

解法#

考虑到操作 F2, 显然应该使用可并堆.

考虑到除了 A2 操作, 其他操作都是基本可并堆操作. 这里只讨论 A2 操作. 不妨考虑对于每个堆, 在堆顶维护一个 $tag$ 表示这个堆的总体增量. 在合并时, 保留大小较大堆顶的 $tag$, 暴力修改大小较小堆顶的 tag.

ODT#

维护序列信息, 基于随机性. 要求操作中有区间推平.

具体来讲, 维护一个区间的集合. 在一个区间被推平时, 我们将其所有位置拆出来, 然后新建一个新区间维护这些位置的信息.

采用 std::set 维护这些区间.

进阶 (?) 内容#

树状数组#

单点修改, 前缀和.

对于任意一个序列 $a$, 我们不妨考虑这样的一个 $c_n = \sum\limits_{i=1}^{2^k} c_{n-i+1}$, 其中 $k = \texttt{lowbit}(n)$. 考虑 $c$ 构成的树状结构, 即对于数上的节点 $u$, 权值为 $c_u$, 父亲为 $u + \texttt{lowbit}(u)$.容易证明 $c_v = \sum\limits_{u+\texttt{lowbit}(u) = v} c_u$. 因此查询 $\sum\limits_{i=1}^k a_i$ 时, 考虑每次将 $c_k$ 计入答案, 然后令 $k \leftarrow k-2^{\texttt{lowbit}(k)}$. 这样能够覆盖 $[1, k]$ 的所有节点. 修改时, 每次更新 $c_{k}$, 然后令 $k\leftarrow k+2^{\texttt{lowbit}(k)}$.

常用 trick : 倍增砍老哥 ($\log$)

线段树#

万恶之源.

区间修改, 区间求和.

对于 $x, y$, 定义 $h(x, y)$ 为最大的 $\frac{p}{q}$ 满足 $xp \leq yq$.
